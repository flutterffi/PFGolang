# Exercises

Use these prompts when you want hands-on practice instead of only reading examples.

## Foundations

1. Change `foundations/02_variables_and_constants` to print your own name and target learning level.
2. Update `foundations/04_conditionals` to add one more score range.
3. Change `foundations/05_loops` to print numbers in reverse order.
4. Add one more function to `foundations/06_functions`.
5. Turn `foundations/08_structs` into a study tracker with at least two lessons.
6. Add a second implementation to `foundations/09_methods_and_interfaces`.

## Intermediate

1. Add one more field to `pkg/tracker.Session`.
2. Write a new test case in `pkg/calc/calc_test.go`.
3. Extend `intermediate/16_json_basics` with a nested field.
4. Update `intermediate/17_file_io` to write two files instead of one.
5. Change `intermediate/18_http_server` to return a second JSON field.

## Modules

1. Create a new module under `modules/05_your_own_module`.
2. Add it to `go.work`.
3. Run `go work sync`.
4. Make sure `go run ./modules/05_your_own_module` works.

## Advanced

1. Add a new benchmark to `advanced/26_benchmarks`.
2. Create a new fake implementation for `advanced/29_service_tests_with_mocks`.
3. Extend `advanced/36_small_rest_api_project` with a new route.
4. Make `advanced/42_full_todo_api_project` reject empty titles.
5. Add a second feature flag to `advanced/48_production_style_todo_service`.
6. Extend `advanced/54_deployment_ready_todo_service` with a health endpoint.

## Stretch Work

1. Add table-driven tests to one advanced package that currently has no tests.
2. Replace one in-memory store with file-based persistence.
3. Add request logging middleware to one todo service example.
4. Build `advanced/55_your_own_service` using the same repo style.
